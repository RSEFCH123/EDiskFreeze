# EDiskFreeze

一个基于 Windows 驱动层的重启还原软件：每次开机时把 **系统所在的物理硬盘** 恢复到设定的基准状态，用来保护公共电脑、网吧、教学机房或个人测试环境。

---

## 它做什么

- **重启即还原**：对系统所在物理硬盘开启还原后，任何写入操作（安装软件、修改系统设置、写入文件）只保存在易失层，重启后自动回到基线状态。
- **绑定系统硬盘**：本软件锁定保护的是「系统所在的物理硬盘」，即安装 Windows 时 C 盘所在的那块物理盘，不支持用户自由选择任意分区。
- **配套控制台**：通过官方 `EDiskFreezeU.exe` 管理还原开关、修改密码、查看状态；所有变更走驱动 IRP 通道。

---

## 密码机制（请仔细阅读）

为了避免产生误导，这里把密码的运作方式如实写清楚：

- **存储方式**：密码以双层哈希的形式落到配置中，先在用户态做一次哈希，再到驱动层做一次哈希后写入，配置文件里没有原始密码的可逆信息。
- **校验位置**：所有受保护操作（开关还原、修改密码等）都通过驱动层的 IRP 派发口完成，密码校验在驱动内进行。
- **没密码时，驱动只认「校验通过」这一条**：在没有正确密码的情况下，驱动不认任何程序——包括官方程序在内的所有请求都会被拒绝。
- **有密码时，第一层哈希在哪一个程序里都通过**：第一层哈希是在用户态完成的，因此一旦拿到正确密码、完成了第一层哈希，关闭还原这一步本身不限定必须是官方程序——任何能在用户态算出第一层哈希、并通过驱动层第二层校验的程序，都能执行关闭还原等操作。换句话说，**密码才是真正的钥匙，软件本身不是**。
- **修改密码的特殊逻辑**：在设置界面修改密码时，必须先关闭系统硬盘的还原并重启；运行期间即使通过底层手段写入了新的密码数据，重启时也会被驱动层按基线状态回滚，无法真正生效。

需要说明的是，「拿到密码就等于拿到控制权」是这类还原软件的共性。如果用 PE 等离线环境直接读写存储介质，理论上也可以对数据进行干预；这并不是本软件独有的问题。本软件能保证的是：在正常运行的 Windows 环境下，**没有密码就关不掉还原、改不了设置**。

## 性能与硬件影响

由于每次写入都需要经过驱动的易失层处理（写入重定向、回滚跟踪等），**会显著消耗硬盘 I/O**，在大量磁盘写入场景下可能出现：

- 顺序/随机读写速度下降
- 大量小文件操作时延迟上升
- 编译、安装大型软件、磁盘基准测试等场景下尤为明显

对老旧机械硬盘（HDD）和低端固态硬盘（SSD）的影响会更明显。建议在性能要求较高的场景中谨慎评估。

## 运行环境

- 操作系统：Windows 10 / Windows 11（x64）
- 架构：x64
- 需要管理员权限安装驱动和注册服务
- **测试环境**：本软件主要在虚拟机中完成功能测试，**未在物理机上做过严格的全场景兼容性验证**。在物理机（尤其是带 BitLocker、RAID、NVMe 特殊工作模式的机器）上首次部署前，请先在非生产环境充分验证。

---

## 快速开始

1. 在 [Releases](../../releases) 页面下载最新版本的安装包。
2. 以管理员身份运行安装程序，按向导完成驱动和服务的部署。
3. 在设置界面确认系统硬盘已被识别，点击「启用还原」锁定整块物理硬盘。
4. 重启一次电脑，还原机制即生效。

之后每次重启，系统所在物理硬盘都会自动回到你启用时的状态。
